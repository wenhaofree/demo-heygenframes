# 中文分镜

**格式：** 1920x1080  
**语言：** 简体中文  
**总时长：** 5 分钟  
**风格：** 深色科技信息图、关系图动画、卡片式高级解释  
**核心母题：** 工具 / 模型 / 厂商 三层关系反复出现

## 节奏

1. **Beat 1 / 0:00 - 0:10**
   主题：名字先砸过来，观众先感受到“乱”  
   画面：大量 AI 名词飞入，形成混乱词云，右侧三层标签待命。

2. **Beat 2 / 0:10 - 0:40**
   主题：建立最重要的判断公式  
   画面：工具、模型、厂商三层关系轨道逐步点亮，右侧出现“三个问题”方法论卡。

3. **Beat 3 / 0:40 - 1:20**
   主题：OpenAI 体系  
   画面：OpenAI 公司卡、GPT 模型核心、ChatGPT / Codex 入口卡同时出现并串联。

4. **Beat 4 / 1:20 - 2:00**
   主题：Anthropic 体系  
   画面：Anthropic 公司卡、Claude 模型核心、Claude / Claude Code 入口卡，强调“不是 Cloud Code”。

5. **Beat 5 / 2:00 - 2:40**
   主题：Google / Gemini 体系  
   画面：Google 公司卡、Gemini 模型核心、Gemini App / AI Studio / API 入口卡，突出 Gemini 双重身份。

6. **Beat 6 / 2:40 - 3:50**
   主题：国内三家  
   画面：Kimi、DeepSeek、GLM 三列系统卡同时展开，逐列高亮解释命名差异。

7. **Beat 7 / 3:50 - 4:30**
   主题：总表总结  
   画面：一张横向总表统一呈现体系、模型、厂商、入口，扫描线逐行高亮。

8. **Beat 8 / 4:30 - 5:00**
   主题：方法论收口  
   画面：三个大问题卡放大停留，背景关系节点收束，最后落到“关注文浩 AI”。

## 视觉重点

- 核心信息保持在画面中间 1200px 区域，方便后续裁成竖版。
- 不用真实 Logo，不用真实人物脸，只用文字、字母标记、色带、抽象节点。
- 背景始终保留缓慢流动的网格、光晕、扫描条和幽灵大字，避免纯静态。
- 三层关系在每个章节都要反复出现，让观众记住“拆法”，而不是记住一堆散名词。

## 关键画面说明

- 名词云：Codex、Claude Code、Gemini、Kimi、DeepSeek、GLM、OpenAI、Anthropic、Google、月之暗面、智谱 AI
- 认知动作：先制造混乱，再露出分类框架

- 工具层：ChatGPT、Codex、Claude Code、Gemini App、Kimi、DeepSeek App、Z.ai

- 厂商层：OpenAI、Anthropic、Google、Moonshot AI、DeepSeek、智谱 AI / Z.ai

- OpenAI ← GPT ← ChatGPT / Codex
- 强调：同一家公司可以做多个入口

### Beat 4 / Anthropic

- Anthropic ← Claude ← Claude / Claude Code

- Google ← Gemini ← Gemini App / AI Studio / API

- DeepSeek ← DeepSeek 模型 ← DeepSeek
