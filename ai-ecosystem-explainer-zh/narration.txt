你是不是也经常分不清，Codex 到底是不是模型，Claude Code 是工具还是大模型，Gemini 又到底是产品、模型，还是 Google 的 AI 系列。

再加上 Kimi、DeepSeek、GLM 这些名字一起出现，很多人越看越乱。

这条视频，不聊谁最强，先帮你把工具、模型、厂商这三层关系一次性理顺。

先记住一个最重要的判断公式。

你看到的名字，可能是工具，也可能是模型，也可能是公司品牌。

工具，是你直接使用的入口。

模型，是负责理解、推理和生成内容的大脑。

厂商，是开发、训练、运营这些模型和产品的公司。

所以以后看任何一个 AI 名字，先问三个问题。

我在哪里用它。

它背后调用什么模型。

这个模型又是哪家公司做的。

先看海外三大体系。

第一组是 OpenAI。

普通用户最熟悉的是 ChatGPT，开发者更常提到 Codex。

如果用这条线去理解，OpenAI 是厂商，GPT 五点五 是模型，ChatGPT 和 Codex 是不同使用入口。

同一个厂商，可以把同一个模型能力，包装成不同产品形态。

第二组是 Anthropic。

它最核心的产品品牌是 Claude。

而 Claude Code，更像是面向开发者和工程任务的入口。

所以这里同样要拆开看。

Anthropic 是公司，Claude Opus 四点八 是模型，Claude 和 Claude Code 是你实际接触到的产品界面或工具。

第三组是 Google。

Gemini 这个名字最容易让人误会，因为它同时像产品名，也像模型名。

更容易理解的方法是，Google 是公司，Gemini 三点五 Flash 和 Pro 是模型系列，Gemini App、AI Studio 和 API 是不同入口。

普通用户常看到 App，开发者更多看到 Studio 和 API。

再看国内三组常见体系。

第一组是 Kimi。

很多人知道 Kimi，但不一定马上能说出背后公司叫月之暗面，也就是 Moonshot A I。

如果把这条线拆开，Kimi 客户端、网页版和 API 是入口，Kimi K 二点六 或 K 二点七 Code 是模型，Moonshot A I 是厂商。

第二组是 DeepSeek。

它比较特别，因为 DeepSeek 既是公司名，也经常直接被当成产品品牌。

所以更要拆。

你在网页、App 或 API 里接触到的是入口，DeepSeek V 四 Pro 和 V 四 Flash 是模型，而做这些模型和产品的公司也叫 DeepSeek。

第三组是 G L M。

G L M 背后对应的是智谱 A I，现在也经常看到 Z 点 A I 这个品牌表达。

如果你看到的是网页版、插件、API，甚至 Auto G L M 这类工具，那是入口。

G L M 五点一 是模型。

智谱 A I 或 Z 点 A I，是厂商和品牌体系。

把这几组放进一张总表，你就会发现，很多混乱其实来自名字层级不同。

有些名字本身是入口，比如 Codex、Claude Code、Kimi 客户端。

有些名字本身是模型，比如 GPT 五点五、Claude Opus 四点八、G L M 五点一。

还有些名字，既是公司品牌，又经常被大家拿来直接代指整套产品，比如 Gemini、DeepSeek、Kimi。

所以以后你再看到一个 A I 名字，不要先急着问它强不强。

先问，它是工具、模型，还是公司。

再问，它背后调用什么模型。

最后问，这个模型是谁做的。

只要这三步走通，很多新名词你第一次见，也能自己拆开。
