# 中文解说脚本

## 目标

用一条 5 分钟以内的中文信息图动画，讲清楚 Codex、Claude Code、Gemini、Kimi、DeepSeek、GLM 这些名字到底分别属于工具、模型还是厂商。

## 旁白分段

### Beat 1 / 开场钩子

你是不是也经常分不清，Codex 到底是不是模型，Claude Code 是工具还是大模型，Gemini 又到底是产品、模型，还是 Google 的 AI 系列。  
再加上 Kimi、DeepSeek、GLM 这些名字一起出现，很多人越看越乱。  
这条视频，不聊谁最强，先帮你把工具、模型、厂商这三层关系一次性理顺。

### Beat 2 / 三层认知

先记住一个最重要的判断公式。  
你看到的名字，可能是工具，也可能是模型，也可能是公司品牌。  
工具，是你直接使用的入口。  
模型，是负责理解、推理和生成内容的大脑。  
厂商，是开发、训练、运营这些模型和产品的公司。  
所以以后看任何一个 AI 名字，先问三个问题。  
我在哪里用它。  
它背后调用什么模型。  
这个模型又是哪家公司做的。

### Beat 3 / 海外三大体系

先看海外三大体系。  
第一组是 OpenAI。  
普通用户最熟悉的是 ChatGPT，开发者更常提到 Codex。  
如果用这条线去理解，OpenAI 是厂商，GPT-5.5 是模型，ChatGPT 和 Codex 是不同使用入口。  
同一个厂商，可以把同一个模型能力，包装成不同产品形态。

第二组是 Anthropic。  
它最核心的产品品牌是 Claude。  
而 Claude Code，更像是面向开发者和工程任务的入口。  
所以这里同样要拆开看。  
Anthropic 是公司，Claude Opus 4.8 是模型，Claude 和 Claude Code 是你实际接触到的产品界面或工具。

第三组是 Google。  
Gemini 这个名字最容易让人误会，因为它同时像产品名，也像模型名。  
更容易理解的方法是，Google 是公司，Gemini 3.5 Flash 和 Pro 是模型系列，Gemini App、AI Studio 和 API 是不同入口。  
普通用户常看到 App，开发者更多看到 Studio 和 API。

### Beat 4 / 国内三大体系

再看国内三组常见体系。  
第一组是 Kimi。  
很多人知道 Kimi，但不一定马上能说出背后公司叫月之暗面，也就是 Moonshot AI。  
如果把这条线拆开，Kimi 客户端、网页版和 API 是入口，Kimi K2.6 或 K2.7 Code 是模型，Moonshot AI 是厂商。

第二组是 DeepSeek。  
它比较特别，因为 DeepSeek 既是公司名，也经常直接被当成产品品牌。  
所以更要拆。  
你在网页、App 或 API 里接触到的是入口，DeepSeek-V4-Pro 和 V4-Flash 是模型，而做这些模型和产品的公司也叫 DeepSeek。

第三组是 GLM。  
GLM 背后对应的是智谱 AI，现在也经常看到 Z.ai 这个品牌表达。  
如果你看到的是网页版、插件、API，甚至 AutoGLM 这类工具，那是入口。  
GLM-5.1 是模型。  
智谱 AI 或 Z.ai，是厂商和品牌体系。

### Beat 5 / 总表总结

把这几组放进一张总表，你就会发现，很多混乱其实来自名字层级不同。  
有些名字本身是入口，比如 Codex、Claude Code、Kimi 客户端。  
有些名字本身是模型，比如 GPT-5.5、Claude Opus 4.8、GLM-5.1。  
还有些名字，既是公司品牌，又经常被大家拿来直接代指整套产品，比如 Gemini、DeepSeek、Kimi。  
所以以后你再看到一个 AI 名字，不要先急着问它强不强。  
先问，它是工具、模型，还是公司。  
再问，它背后调用什么模型。  
最后问，这个模型是谁做的。  
只要这三步走通，很多新名词你第一次见，也能自己拆开。

### Beat 6 / 结尾

记住今天这条最有用的方法。  
看见 AI 名字，先分层，再评价。  
先分清工具、模型、厂商，再谈能力、价格和场景。
